# Configuration Files

All training runs are configured via YAML files in this directory.

## `lora_training.yaml`

Template for LoRA + reservoir interface training (used as a base by training scripts).
Key sections:

| Section | Description |
|---------|-------------|
| `lora_*` | LoRA rank, alpha, dropout, target modules |
| `lora_lr` / `interface_lr` | Separate learning rates for LoRA vs reservoir interface |
| `num_training_steps` | Total optimizer steps |
| `gradient_checkpointing` | Enable memory-efficient gradient checkpointing |
| `dataset_mixing_ratios` | Weights for FineWeb / synthetic / chaotic data mix |

## `track_a_readonly.yaml`

Track A read-only phase: frozen Qwen3.5-0.8B + LoRA + ESN sidecar (no WriteHead).
Used by `scripts/train_track_a_readonly.py`.

Key parameters:
- `reservoir_size`: Number of ESN nodes (default 10000)
- `spectral_radius`: Should be < 1 for echo state property (default 0.9)
- `leak_rate`: 0 = very long memory, 1 = memoryless (default 0.5)
- `sidecar_layers`: Which transformer layers to inject the sidecar (null = auto)

## `sweep/`

Auto-generated sweep configs for the reservoir hyperparameter sweep (Task T16).
Generated by `scripts/sweep_reservoir_hp.py`.

Each file sweeps one dimension while holding others at their default values:

| File | Dimension | Values |
|------|-----------|--------|
| `size_500.yaml` | `reservoir_size` | 500 |
| `size_2000.yaml` | `reservoir_size` | 2000 |
| `size_10000.yaml` | `reservoir_size` | 10000 |
| `size_50000.yaml` | `reservoir_size` | 50000 |
| `leak_rate_0p1.yaml` | `leak_rate` | 0.1 |
| `leak_rate_0p3.yaml` | `leak_rate` | 0.3 |
| `leak_rate_0p7.yaml` | `leak_rate` | 0.7 |
| `leak_rate_1p0.yaml` | `leak_rate` | 1.0 |
| `spectral_radius_0p5.yaml` | `spectral_radius` | 0.5 |
| `spectral_radius_0p9.yaml` | `spectral_radius` | 0.9 |
| `spectral_radius_0p99.yaml` | `spectral_radius` | 0.99 |
| `spectral_radius_1p1.yaml` | `spectral_radius` | 1.1 |
| `topology_erdos_renyi.yaml` | `topology` | erdos_renyi |
| `topology_small_world.yaml` | `topology` | small_world |
| `best_combo.yaml` | (best from sweep) | see Pareto analysis |

Private fields prefixed with `_sweep_*` are metadata added by the sweep runner.

## Running a sweep

```bash
# Run all 14 sweep configs sequentially
python scripts/sweep_reservoir_hp.py

# Results are written to results/track_a/sweep/
# Pareto frontier: results/track_a/sweep/pareto_frontier.json
# Gate A analysis: python scripts/gate_a_analysis.py
```
